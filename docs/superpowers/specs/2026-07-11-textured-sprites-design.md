# Textured Sprites — Design Spec

**Date:** 2026-07-11
**Track:** B (Mini Studio) — sub-project 3, the join between Texture Lab (v1) and Sandbox (v1).
**Status:** approved (standing auto-approve), implementing.

## 1. Goal

A sandbox actor can wear a texture generated by the Texture Lab instead of a flat
colour. This closes the loop between the two Mini-Studio tools already shipped:
`--studio` *produces* `.hrt` textures; `--sandbox` now *consumes* them. One new
data field, one new renderer primitive, one discovery probe — no new subsystem.

## 2. Non-goals (deferred)

- Rotated texture blit (the sim has `rot`, but we blit axis-aligned; a rotated
  resampler is a renderer feature, not a sandbox one). Orientation tick still shows spin.
- Per-actor texture *tinting* / atlasing / animation frames.
- Directory scan of the texture collection (`<filesystem>`, native-only). v1 probes a
  fixed name range that matches the Texture Lab's own save naming (§6). Upgrade path noted.
- Editing textures from inside the sandbox — that stays the Texture Lab's job.

## 3. Constraints honoured

- **Pure core stays pure.** `sandbox_core` gains no image/IO code — the texture is a
  *name* (a string), resolved to pixels only in the SDL-touching scene. Headless tests
  keep working on plain strings.
- **Every pixel is ours.** The scaled blit is hand-written nearest-neighbour; no SDL
  drawing. It lives in `Renderer2D` as a general primitive (all games can use it).
- **Web-portable discovery.** No `<filesystem>`; the scene probes asset names through the
  existing `assets::` seam, which works native and on the WASM virtual FS.

## 4. Data model (pure core, `world.hpp`)

Add one field to the two structs that describe an actor's look:

```cpp
struct Sprite    { gfx::Color color = ...; bool round = false; std::string texture; };
struct Archetype { ...; std::string texture; };   // "" = flat colour (unchanged default)
```

`World::spawn` copies `a.texture` into the new `Sprite`. Empty string = today's flat
behaviour, so every existing actor and test is byte-identical.

## 5. Serializer (`serialize.cpp`)

One token, emitted only when set, so untextured scenes are unchanged on disk:

- `archetype_tokens`: append ` tex=<name>` when `!a.texture.empty()`.
- `parse_archetype`: `k == "tex"` → `a.texture = v`.

Texture names are `studio_NN` (no spaces/brackets), so the existing bracket-aware
tokenizer needs no change. Round-trips through save/load **and** the Play/Stop snapshot
for free (the snapshot is a scene string).

## 6. Texture discovery (scene)

The Texture Lab saves to `textures/studio_%02d.hrt` (`studio_scene.cpp:50-52`). The
sandbox discovers what exists by probing that exact range once, on first render:

```
for i in 0..31:  load_image("textures/studio_%02d.hrt")  -> keep those that decode
```

Kept images go into `std::unordered_map<std::string, gfx::Image> tex_` (name→pixels)
plus an ordered `std::vector<std::string> tex_names_` for cycling. `load_image` returns
`nullopt` for a missing/garbage file, so the probe is self-limiting and cross-platform.

> ponytail: 32-name probe, not a dir scan — matches the Lab's naming and needs no
> `<filesystem>`. Swap in a manifest/dir-scan when the collection outgrows 32.

## 7. Rendering (`sandbox_scene.cpp` draw view)

```
if s.texture non-empty and tex_ has it:  blit_scaled(image, cx-dw/2, cy-dh/2, dw, dh)
else:                                     current flat fill (circle if round, else rect)
```

`dw,dh` already fold in `Body` size × `Transform2D.scale`, so a textured actor stretches
with the scale slider exactly like a flat one. Selection outline and orientation tick
draw on top, unchanged.

## 8. Renderer primitive (`renderer2d.hpp/.cpp`)

```cpp
void blit_scaled(const Sprite& s, int dx, int dy, int dw, int dh);
```

Nearest-neighbour resample of the source image into the logical `dw×dh` rect, alpha
skipped on fully-transparent source texels, everything clipped, scaled by `ss_` like the
existing `blit`. No-op on null pixels or non-positive `dw/dh/s.w/s.h`.

## 9. Assignment UI (inspector)

When textures exist, the inspector gains a **`Texture: <name|none>`** button that cycles
`none → tex_names_[0] → … → none`, writing the chosen name into the selected actor's
`Sprite.texture`. Recolor still works (it just isn't visible while a texture covers the
fill). When no textures were found, the button is hidden and a muted hint points at
`--studio`.

## 10. Tests

Pure-core (`tests/test_sandbox.cpp`), all headless:

1. **spawn_copies_texture** — `Archetype.texture` lands on the spawned `Sprite`.
2. **archetype_codec_texture** — `tex=` survives `archetype_tokens`→`parse_archetype`.
3. **scene_roundtrip_texture** — a textured actor survives `to_scene`→`from_scene`
   (and an untextured actor emits **no** `tex=` token — the on-disk format is unchanged).

Renderer (`tests/test_aa.cpp`):

4. **blit_scaled** — 2×2 source upscaled to 4×4 fills the block; a transparent texel is
   skipped; a 4×4 source downscaled to 2×2 samples corners; out-of-rect pixels stay bg.

## 12. Risks

- **Texture doesn't match Body aspect** → stretch. Acceptable; the Lab makes square
  128² textures and default actors are square-ish. No fix in v1.
- **Probe range too small** (>32 saved textures) → later ones invisible in the sandbox.
  Logged as a ponytail ceiling with the dir-scan upgrade path.

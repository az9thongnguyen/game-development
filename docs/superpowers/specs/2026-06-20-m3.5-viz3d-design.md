Date: 2026-06-20
Status: approved (standing "auto-approve, proceed" directive)

> Decisions record. The *how & why* is taught in the guidebook (`docs/book/24`–`25`).

---

## 1. Goal & scope

Turn the static M3 showcase into an **interactive editor**: create, place, transform,
and delete 3D objects with the mouse and keyboard, with an intuitive **orbit/pan/zoom**
camera. This is the foundation for future visualization tools.

Deliver:
- A **scene of objects**, each = shape + transform (position, rotation, scale) + color.
- **Spawn** primitives (cube / sphere / plane / cylinder).
- **Select** by mouse **picking** (ray vs bounding sphere); Tab to cycle.
- **Transform** the selection: translate (drag on ground), lift (Y), rotate, scale.
- **Delete** the selection.
- **Camera**: orbit (right-drag), pan (middle-drag), zoom (keys), focus selected.
- **Toggles**: shading (wire/flat/Gouraud), grid, axes, backface culling.
- A **selection highlight** (yellow wire overlay) + a HUD.

Out of scope (later): multi-select, undo/redo, gizmo handles, save/load (M4 brings
serialization), textured objects.

Constraints unchanged: hand-written into the CPU framebuffer, SDL only in the platform
layer, web-ready, single-threaded.

---

## 2. New platform capability: a fuller keyset

An editor needs more keys than the M0 demo did. The `Key` enum + the SDL scancode table
in `backend_sdl.cpp` are extended (platform-layer only — nothing leaks upward):

`Num1..Num4, Q, E, R, F, G, C, X, Minus, Equals, Tab, Delete, Backspace`

The pump already loops over the table, so new rows "just work." This is also a small
lesson in the platform seam: input grows in one place.

## 3. Controls

| Input | Action |
|---|---|
| **1 / 2 / 3 / 4** | spawn cube / sphere / plane / cylinder at the camera target (auto-selected) |
| **left-click** | pick the object under the cursor (empty space → deselect) |
| **Tab** | cycle selection to the next object |
| **left-drag** (on selected) | translate it on its ground plane (follows the mouse ray) |
| **Q / E** | move selected down / up (Y axis) |
| **arrows** | rotate selected (yaw/pitch); if nothing selected, orbit the camera |
| **− / =** | scale selected down / up |
| **Delete / Backspace** | delete selected |
| **Enter** | cycle shading: wireframe → flat → Gouraud |
| **G / X / C** | toggle grid / axes / backface culling |
| **F** | focus camera on the selected object (set orbit target) |
| **right-drag** | orbit camera · **middle-drag** pan · **W / S** zoom in / out |
| **Esc** | quit |

## 4. Module layout

```
src/engine/
  pick.hpp            PURE header: ray_sphere(), ray_plane_y()  (testable)
  camera.{hpp,cpp}    + forward/right/up, ray_through(ndc,aspect)->Ray, pan()
  geometry.{hpp,cpp}  + make_cylinder()
  renderer3d.{hpp,cpp}+ draw_wire(mesh, model, color)  (fixed-color edge overlay)
src/games/viz3d/
  editor.{hpp,cpp}    PURE model: Shape, Object (transform+model()+bound_radius()),
                      Editor (objects, selection, spawn/remove/select/cycle/pick)
  editor_scene.{hpp,cpp}  interactive Scene: input -> editor + camera + render + HUD
tests/
  test_viz3d.cpp      pick math, editor model, camera ray (CTest, no SDL)
```

Reuse pattern: `editor.cpp` is pure (no SDL, no renderer) → unit-tested. `editor_scene`
is the thin interactive shell, the only new SDL-touching... actually it touches no SDL
either — it reads `InputState` and draws via `Renderer3D`. Same architecture as every
other scene.

### Key types

```cpp
namespace viz3d {
enum class Shape { Cube, Sphere, Plane, Cylinder };
struct Object {
    Shape shape; math::vec3 pos{}; float yaw=0, pitch=0, scale=1;
    gfx::Color color;
    math::mat4 model() const;       // T * Ry * Rx * S
    float bound_radius() const;     // for picking (generous bounding sphere)
};
class Editor {
    std::vector<Object> objects_; int selected_ = -1;
public:
    int  spawn(Shape, math::vec3 at, gfx::Color);  // appends, selects, returns index
    void remove_selected();
    void select(int i); void cycle();              // Tab
    int  pick(math::vec3 ro, math::vec3 rd) const; // nearest hit, or -1
    Object* selected();  const std::vector<Object>& objects() const;
};
}
```

```cpp
namespace pick {
// nearest positive t where the ray hits the sphere, or -1 if it misses.
float ray_sphere(math::vec3 o, math::vec3 d, math::vec3 c, float r);
// intersect ray with horizontal plane y=py; false if parallel/behind.
bool  ray_plane_y(math::vec3 o, math::vec3 d, float py, math::vec3& out);
}
```

## 5. Picking & dragging (the math)

- **Screen→ray.** `ray_through` builds a world ray from the camera basis:
  `dir = normalize(forward + ndc_x·tan(fovy/2)·aspect·right + ndc_y·tan(fovy/2)·up)`,
  origin = `eye()`. Caller maps the mouse pixel to `ndc = (2·mx/W−1, 1−2·my/H)`.
- **Select.** `Editor::pick` runs `ray_sphere` against every object's world bounding
  sphere (center = `pos`, radius = `bound_radius()`), returns the **nearest** hit.
- **Translate.** While dragging a selected object, intersect the mouse ray with the
  horizontal plane at the object's current `y` (`ray_plane_y`) and set its `x,z`.

No general `mat4` inverse is needed — the camera basis gives the ray directly (simpler
and teachable).

---

## 6. Rendering

`EditorScene::render` composes via the M3 core: floor + grid + axes, then each object's
mesh by shape in the current shading mode, then the **selected** object again via
`draw_wire(mesh, model, yellow)` as a highlight. HUD shows object count, selected
shape/scale, and the control legend.

Meshes are shared **templates** (one cube, one sphere, one plane, one cylinder), each
drawn many times with per-object model matrices — the indexed-reuse idea from Chapter 20
applied at the object level.

## 7. Test plan (`tests/test_viz3d.cpp`, CTest, no SDL)

- `ray_sphere`: head-on ray hits at the expected `t`; a parallel/offset ray misses;
  a ray starting inside returns the far hit.
- `ray_plane_y`: a downward ray hits `y=0` at the expected point; an upward/parallel ray
  reports no hit.
- `OrbitCamera::ray_through(0,0)` points from the eye toward the target.
- `Editor`: spawn appends + selects; `pick` selects the nearer of two objects and
  returns −1 on a miss; `remove_selected` deletes and fixes the selection index;
  `cycle` wraps.
- `make_cylinder` index/vertex counts; existing suites stay green.
- Headless `--viz3d` run exits 0; 0 leaks; no SDL above platform.

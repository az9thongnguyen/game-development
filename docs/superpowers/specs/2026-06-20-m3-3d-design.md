## 1. Goal & scope

Build a **hand-written software 3D renderer** that is a **reusable subsystem**, not
a one-off demo. M3 delivers:

- A **software triangle rasterizer** with a **z-buffer** and **perspective** depth.
- The full **transform pipeline**: model → view → projection → perspective divide →
  viewport (→ screen, with y-flip).
- **`geometry`**: a `Mesh` (vertex + index) plus **primitive generators** — cube,
  plane, grid, sphere, and an axis gizmo.
- **`camera`**: an **orbit camera** (for visualization) and a **free/FPS camera**.
- **Wireframe** and **solid** draw modes; solid supports **flat** and **Gouraud**
  shading with a single directional light.
- **Backface culling** (toggleable) and **near-plane clipping**.
- A **`scene3d` demo** that renders and rotates meshes with correct depth, a ground
  grid, and XYZ axes — the seed for the M3.5 `viz3d` sandbox.

**Out of scope for M3** (deferred): the interactive M3.5 sandbox (object
create/place/transform with the mouse), textured triangles, full 6-plane frustum
clipping, and any OpenGL/WebGL path. We bake the seams; we don't build them yet.

The hard constraints from `requirements.md` §2 still hold: **everything is drawn by
hand into the CPU framebuffer**; SDL stays a thin shim; no blocking loop; web-ready.

## 2. Conventions (locked — these prevent matrix bugs)

These extend the conventions already in `engine/math.hpp`:

- **Right-handed** world; camera looks down **−Z** (matches `mat4_look_at`).
- **Column-major** matrices; transform with `M * v`.
- **Clip space**: `mat4_perspective` maps depth to `[-1, +1]` (near → −1, far → +1).
- **NDC** after perspective divide: `[-1,+1]³`.
- **Viewport**: `mat4_viewport(x,y,w,h)` maps NDC x,y to the window rect and NDC z
  `[-1,+1]` → **depth `[0,1]`** (near = 0, far = 1).
- **Screen y-flip**: the framebuffer has +y **down** (renderer2d convention), but NDC
  +y is **up**. We flip y at the viewport step by passing **negative height** to a
  viewport built for screen space, i.e. `screen_y` decreases as NDC y increases.
  This is the "y-flip handled at rasterization time" promised in `math.hpp`.
- **Depth test**: **smaller depth = nearer.** Z-buffer initialised to `+∞`
  (`1e30f`); a fragment is kept iff `depth < zbuf[i]`, then `zbuf[i] = depth`.
- **Winding**: **counter-clockwise = front face** (OpenGL default). After the y-flip
  the screen-space signed area flips sign, which the culling test accounts for.

## 3. Module layout

All three new modules are **engine core** (reusable), under `src/engine/`. They
write into the *same* framebuffer the 2D renderer uses (requirements §4: "renderer2d
và renderer3d cùng ghi vào một framebuffer").

```
src/engine/
  geometry.hpp / .cpp    Vertex, Mesh, generators (cube/plane/grid/sphere/axes), normals
  camera.hpp  / .cpp     OrbitCamera, FlyCamera  (each -> view(), proj(aspect))
  pipeline.hpp           PURE, header-only: project/edge/barycentric/near-clip/backface
  renderer3d.hpp / .cpp  Renderer3D: owns depth buffer; draw_mesh / draw_wireframe / line3d
src/games/viz3d/
  scene3d.hpp / .cpp     Scene3D demo (rotating meshes + grid + axes + orbit camera)
tests/
  test_render3d.cpp      pure pipeline + geometry unit tests (CTest, dependency-free)
```

**Why split `pipeline.hpp` (pure) from `renderer3d.cpp` (framebuffer)?** Same pattern
as M2's `raycast.cpp` (pure `cast_ray`) vs `raycast_scene.cpp` (the pixel pass): the
math is unit-testable headlessly, the pixel-pushing is not. This is what makes
`test_render3d` possible without a window.

## 4. Rasterization algorithm (decided)

**Edge-function / barycentric rasterizer** over each triangle's screen-space bounding
box (clamped to the framebuffer). Chosen over scanline because it is branch-simple,
handles every triangle orientation uniformly, makes the **top-left fill rule** easy
(no double-drawn shared edges), and the barycentric weights it already computes drive
both depth and attribute interpolation.

Per triangle:
1. Transform 3 verts by `proj * view * model` → clip space (keep world pos + normal
   for lighting).
2. **Near-clip** against `w + z ≥ 0` (Sutherland-Hodgman → up to 2 triangles). This
   removes the `w ≤ 0` divide-by-zero / behind-camera wrap that would otherwise smear
   geometry across the screen.
3. Perspective divide → NDC; viewport (with y-flip) → screen x,y + depth∈[0,1].
4. **Backface cull** (if enabled) using `signed_area` sign.
5. Bounding box, clamped to `[0,W)×[0,H)`. For each pixel, compute barycentric `(w0,
   w1,w2)`; skip if any `< 0` (top-left rule resolves edges).
6. **Depth** = `w0*z0 + w1*z1 + w2*z2` (linear in screen space = correct hyperbolic
   depth). Test/replace against `depth_`.
7. Shade:
   - **Wireframe**: skip fill; draw the 3 edges with `draw_line` (still depth-tested
     optional — M3 draws wireframe without depth for simplicity, documented).
   - **SolidFlat**: one Lambert color from the **face** normal; constant over the tri.
   - **SolidGouraud**: Lambert per vertex, **perspective-correct** interpolation of
     the lit color (interpolate `attr/w` and `1/w`, then divide).

**Lighting**: single directional light, `intensity = ambient + (1-ambient)·max(0,
n·(−lightDir))`, multiplied into the vertex/face albedo. Clamp per channel
(reuse the `shade_color` clamp idea from M2).

## 5. Cameras (decided)

Both produce a `view()` (a `mat4_look_at`) and `proj(aspect)` (a `mat4_perspective`,
fovy ≈ 60°, near 0.1, far 100).

- **OrbitCamera**: state = `target`, `distance`, `yaw`, `pitch`. `eye = target +
  distance · (cosPitch·sinYaw, sinPitch, cosPitch·cosYaw)`. Controls: **left-drag**
  rotates (mouse delta → yaw/pitch, pitch clamped to ±89°), **W/S or Up/Down** zoom
  (distance clamped). For visualization.
- **FlyCamera**: state = `pos`, `yaw`, `pitch`. forward from yaw/pitch; **WASD**
  moves on the ground-relative basis, **arrows** turn (no mouse-capture needed so it
  stays web-friendly). For walking through a scene.

Mouse wheel is **not** in our input snapshot, so zoom is on keys — keeps the platform
seam unchanged and stays portable.

## 6. Demo scene (`scene3d`) & dispatch

`--3d` launches a 960×600 smooth+HiDPI window with `Scene3D`:
- A ground **grid** + **XYZ axes** for orientation.
- A spinning **cube** and a **sphere** (and a flat plane) at distinct positions,
  proving correct mutual occlusion via the z-buffer.
- **Orbit camera** by default; **F** toggles to fly camera.
- Keys toggle: **1** wireframe / **2** flat / **3** Gouraud, **C** backface culling,
  **G** grid, **X** axes. HUD text shows current modes + FPS.

This scene is deliberately the seed of the M3.5 `viz3d` sandbox (interactive object
creation comes next milestone).

## 7. Test plan (`tests/test_render3d.cpp`, CTest, no SDL)

Pure functions make this fully headless:
- **Projection**: a known point through a known MVP+viewport lands at the expected
  screen pixel and depth; a centered point projects to screen center.
- **Backface**: a CCW triangle is front-facing, its CW reverse is back-facing
  (`signed_area` sign flips).
- **Near-clip**: a triangle fully in front → 1 tri unchanged; fully behind → 0;
  straddling → 1–2 tris with all `w+z ≥ 0`.
- **Edge/barycentric**: weights at the centroid are ~`1/3` each and sum to 1; a point
  outside has a negative weight.
- **Depth compare**: nearer depth wins (simulate two fragments at one pixel).
- **Geometry**: `make_cube` has 8 unique positions / 12 triangles (36 indices);
  `make_grid`/`make_axes` have even index counts (line lists); sphere index count
  matches `stacks*slices*6`; `recompute_normals` yields unit-length normals pointing
  outward on the cube.

Plus existing suites must stay green; a headless `--3d` run (`HAND_ENGINE_FRAMES=N`)
must exit 0; `leaks` must report 0; grep must confirm no `SDL_`/`#include <SDL` above
`src/platform/`.

## 8. Build order (commit per step)

1. Spec + branch. *(this)*
2. `pipeline.hpp` (pure) + `test_render3d` (projection/edge/clip/backface/depth). CTest green.
3. `geometry` (mesh + generators + normals) + geometry tests.
4. `renderer3d` rasterizer: z-buffer solid-flat fill + wireframe + `draw_lines`.
5. `camera` (orbit + free) + camera tests.
6. Shading: Gouraud + backface culling toggles wired into `draw_mesh`.
7. `scene3d` + `--3d` dispatch; manual + headless run.
8. Guidebook ch.18–23; update `README.md` + `00-overview.md`.
9. Review → test → fix loop; merge `--no-ff` into `main`.
